# 10. Developer Contracts

Agreements and responsibilities that define how we work together.

## Local Testing Contract

**As a developer, I commit to:**

- Testing my code locally before pushing
- Running all relevant tests locally
- Ensuring my code doesn't break existing functionality
- Verifying my changes work in the expected environment

**The team commits to:**

- Providing clear testing instructions
- Maintaining a working local development environment
- Documenting test requirements

## CI Responsibility Contract

**As a developer, I commit to:**

- Not pushing code that will break CI
- Fixing CI failures promptly
- Understanding why CI failed before asking for help
- Not merging code with failing CI checks

**The team commits to:**

- Maintaining a stable CI environment
- Providing clear error messages
- Responding to CI issues promptly

## Staging Promotion Contract

**As a developer, I commit to:**

- Ensuring code works in staging before requesting production promotion
- Verifying staging deployments
- Testing critical paths in staging
- Communicating any staging issues

**The team commits to:**

- Maintaining staging as a production-like environment
- Providing staging access to all developers
- Monitoring staging deployments
